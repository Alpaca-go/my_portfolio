## 修复：返回文件夹动画顺序 — 第二版（16:05）
- 需求：返回时两端卡先渐出，中间3张保持可见并按飞出路径原路返回到文件夹。
- 根因：
  1. `.brand-detail.is-closing .brand-detail__fan-card` 让所有 fan-card（含中间3张）0ms 硬切消失，导致原生卡片先消失、过渡层再"凭空出现"飞回，视觉上不是连续的原路返回。
  2. JS 中 `handleDetailBack` 没有分阶段：is-closing、is-card-transitioning、transitionActive 几乎同时触发。
- 改动：
  - `src/styles.css`：新增 `.brand-detail.is-closing:not(.is-card-transitioning) .brand-detail__fan-card--motion-target { opacity:1; transition: opacity 0ms }`，确保阶段A（两端渐出期间）中间3张保持可见；`.brand-detail.is-closing .brand-detail__fan-card--edge` 已有 `opacity 300ms ease` 渐出（会话前本地修改）。
  - `src/App.jsx` `handleDetailBack`：拆为两阶段。阶段A（0-300ms）仅设 `is-closing`，edge 渐出、motion-target 保持可见；阶段B（300ms）设 `detailVisible=false` + `is-card-transitioning` + 启动过渡层飞回；清理顺延到 1000ms。`edgeFadeMs = 300` 与 CSS 同步。
- 验证：Vite HMR 编译通过，dev server 在 http://localhost:5174/ 运行。
